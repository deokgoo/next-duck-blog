## 1. 소개
SEO(검색 엔진 최적화)는 웹사이트가 검색 엔진 결과 페이지(SERP)에서 높은 순위를 차지하도록 도와줍니다. 높은 순위를 차지하면 더 많은 트래픽을 유도할 수 있어 웹사이트의 성공에 중요한 역할을 합니다. 프론트엔드 개발자는 웹사이트의 구조와 성능에 직접적인 영향을 미치기 때문에 SEO를 위한 최적화 작업에서 중요한 역할을 합니다. 특히나 SEO를 잘 수행했을 때와 그렇지 않았을 때의 차이는 매우 큽니다.

### SEO의 중요성
SEO를 제대로 수행하면, 웹사이트의 방문자 수가 급격히 증가합니다. 예를 들어, 키워드 최적화를 통해 특정 검색어로 상위에 노출되면, 그 검색어를 입력한 사용자가 우리 웹사이트를 방문할 확률이 높아집니다. 이는 곧 매출 증대, 브랜드 인지도 향상, 사용자 참여도 증가로 이어질 수 있습니다. 반대로, SEO를 소홀히 하면 웹사이트는 검색 결과에서 묻히게 되어 잠재 고객을 놓치는 결과를 초래합니다.

### SEO의 영향
프론트엔드 개발자는 웹사이트의 구조와 성능에 직접적인 영향을 미치기 때문에 SEO를 위한 최적화 작업에서 중요한 역할을 합니다. 단순히 예쁘게 보이는 웹사이트를 만드는 것을 넘어, 검색 엔진이 우리 웹사이트를 잘 이해하고 인덱싱할 수 있도록 도와야 합니다.

### 접근 방법
SEO를 위한 접근 방법은 크게 두 가지로 나눌 수 있습니다. 기술적 SEO와 콘텐츠 SEO.

**기술적 SEO**는 웹사이트의 성능, 모바일 최적화, 구조화된 데이터 사용, HTML 태그 최적화 등을 포함합니다. 이러한 요소들은 검색 엔진이 웹사이트를 크롤링하고 인덱싱하는 데 중요한 역할을 합니다.

**콘텐츠 SEO**는 키워드 리서치, 고품질 콘텐츠 작성, 내부 링크 구조 최적화 등을 포함합니다. 이는 검색 엔진이 웹사이트의 내용을 이해하고 사용자에게 관련성 있는 정보를 제공하도록 돕습니다.

프론트엔드 개발자는 기술적 SEO에 큰 영향을 미칠 수 있습니다. 페이지 로딩 속도를 최적화하고, 반응형 웹 디자인을 구현하며, 구조화된 데이터를 사용하는 것 등이 이에 해당합니다. 또한, 콘텐츠 SEO를 지원하기 위해 키워드가 잘 반영된 HTML 태그와 메타 데이터를 작성하는 것도 중요합니다. 또, 기술적 SEO를 중요시하다 콘텐츠 SEO를 놓치는 경우도 있는데 이를 유의해서 살펴보자.

## 2. 페이지 로딩 속도 최적화
페이지 로딩 속도는 SEO에 매우 중요한 요소입니다. 검색 엔진은 빠르게 로드되는 페이지를 선호하며, 사용자도 마찬가지입니다.

이미지 최적화
이미지는 웹 페이지의 로딩 속도를 크게 좌우합니다. 이미지 크기를 최적화하고, 필요한 경우 WebP, AVIF 와 같은 최신 이미지 형식을 사용하는 것이 좋습니다. 또한, 이미지의 width와 height 속성을 명시하여 레이아웃 이동을 방지해야 합니다.

코드 스플리팅
코드 스플리팅은 큰 자바스크립트 파일을 작은 청크로 나누어 필요한 시점에 로드하는 기술입니다. 이를 통해 초기 로딩 속도를 향상시킬 수 있습니다. 번들러를(Webpack, Vite, SWC 등) 사용하면 코드 스플리팅을 쉽게 구현할 수 있는데 라이브러리를 분리하고 CDN을 사용하는것도 하나의 방법입니다. 

캐싱 전략
캐싱을 통해 반복되는 네트워크 요청을 줄일 수 있습니다. 브라우저 캐싱, 서비스 워커 등을 활용하여 정적파일을 캐싱하면 성능을 크게 향상시킬 수 있습니다.

AMP 사용 고려
구글이 주도하여 개발한 오픈소스 프로젝트 `AMP`를 사용하면 속도는 물론 구글 플랫폼에서 페이지를 우선적으로 다루는 경우가 많습니다.
단, 제한적인 사용으로 간단한 페이지에만 적용할 수 있습니다. nextjs와 같은 프레임워크에서도 해당 옵션을 지원합니다.
