# 한글 AI 티 제거 — 블로그 글 검토·윤문 가이드

블로그 글에서 AI가 쓴 듯한 문체·표현을 탐지하고, 내용은 보존한 채 자연스러운 한국어로 윤문한다.

## 철칙

1. **의미 불변** — 사실·주장·수치·고유명사·인용은 100% 원문 보존
2. **근거 기반** — 아래 패턴에 해당하는 구간만 수정. 해당 없으면 건드리지 않음
3. **장르 유지** — 블로그 톤을 유지. 문학체·격식체로 바꾸지 않음
4. **과윤문 금지** — 변경률 30% 초과 시 경고, 50% 초과 시 중단

## Do-NOT (수정 금지)

- 고유명사·제품명·모델명
- 수치·날짜·단위
- 큰따옴표 안 직접 인용
- 코드 블록 내용
- 영어 약어(LLM·GPU·API 등 업계 표준)

## 작업 절차

1. 대상 글을 읽고 아래 패턴을 스캔
2. 발견된 패턴을 (ID, 해당 구간, 심각도) 형태로 정리
3. D(관용구) → A(번역투) → I(형식명사) → H(접속사) → F(수식) → E(리듬) 순서로 윤문
4. 자체검증 6항 점검
5. 결과를 before/after 비교와 함께 제시

## 자체검증 체크리스트 (윤문 후 점검)

1. 고유명사·수치·인용 100% 보존됐는가
2. 변경률 30% 이하인가
3. 블로그 톤에서 이탈하지 않았는가
4. S1 패턴이 잔존하지 않는가
5. 원문에 없던 비유·수사를 추가하지 않았는가
6. 코드 블록·기술 용어를 건드리지 않았는가

## 장르별 문체 기준

### 기술 글 (dev·web 카테고리)
- **문어체 존댓말** 기반 — `~했습니다`, `~입니다`, `~합니다`
- `~됐습니다`, `~체감하게 됐습니다` 같은 **회화체 종결** 지양
  - ❌ "점점 더 체감하게 됐습니다"
  - ✅ "점점 더 체감합니다"
- 서론이 두 문단으로 뚝 끊기지 않도록 자연스럽게 연결

### 회고·일상 글 (daily·회고 성격)
- 1인칭 경험 서술 자유롭게 허용
- `~됐습니다`, `~했는데요` 같은 회화체 허용

## 등급

- **A**: S1 잔존 0, S2 ≤2, 변경률 10~25%
- **B**: S1 잔존 0, S2 ≤4
- **C**: S1 잔존 1~2 → 추가 윤문 필요
- **D**: S1 3건+ → 전면 재작업

## 출력 형식

윤문 완료 후 다음을 제시:
1. 한 줄 상태: `완료. 변경률 X% / 등급 Y / 자체검증 N/6 통과`
2. 주요 변경 3~5건 (before → after)
3. 윤문본 전문
