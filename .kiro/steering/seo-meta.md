# SEO 메타 작성 가이드

글을 쓰거나 수정할 때 title, summary, tags, images 필드를 최적화하는 기준.  
(기존 글 점검은 `#seo-audit` 참고)

---

## title

- **길이**: 40~60자. 70자 초과 시 검색 결과에서 잘림
- **구조**: `핵심 키워드 + 부가 설명` — 키워드를 앞에 배치
- 블로그명은 Next.js template이 자동으로 `| Duck Blog` 추가하므로 title에 넣지 않음

```
✅ "React Suspense 진화 과정과 React 19 디자인 패턴"
✅ "HTTP/2 탄생 배경과 핵심 기술 이해하기"
✅ "[카파도키아 숙소] 아르테미스 동굴 호텔 솔직 후기"
❌ "React Suspense에 대해 알아보겠습니다"  ← AI 교과서 톤
❌ "이번 포스팅에서는 HTTP/2를 다룹니다"   ← 검색 키워드 없음
```

---

## summary (description)

- **길이**: 80~160자. 160자 초과 시 snippet에서 잘림
- 글에서 얻을 수 있는 것을 구체적으로 — 추상적인 소개 금지

```
✅ "APLN 오류를 겪으며 HTTP/2를 제대로 공부했습니다.
    HOL 블로킹, Google SPDY, 4가지 핵심 기술까지 정리했습니다."

✅ "카파도키아 동굴 호텔 아르테미스 내돈내산 후기.
    플랫폼보다 저렴하게 투어 예약하는 방법도 함께 정리했습니다."

❌ "HTTP/2에 대해 깊이 있게 알아보는 글입니다."
```

---

## tags

- 3~5개 권장
- 검색에 쓰일 구체적인 단어 포함

```
❌ ["react", "web", "frontend"]          ← 너무 일반적
✅ ["React Suspense", "RSC", "서버 컴포넌트", "Next.js 19"]
```

---

## images

- `images[0]`이 OG/Twitter 카드 이미지로 사용됨 (권장: 1200×630px)
- 없으면 `socialBanner` 기본값으로 대체됨
- 글에 핵심 이미지가 있다면 `images: ["{url}"]`로 명시 권장

## 카테고리별 키워드 방향

| category | 키워드 예시 |
|---|---|
| `web` / `dev` | "~란", "~하는 법", "~차이", "~원리", "~패턴", "~최적화" |
| `travel` | "~추천", "~후기", "~꿀팁", "~방법" |
| `daily` | 키워드 최적화보다 자연스러운 제목 우선 |
