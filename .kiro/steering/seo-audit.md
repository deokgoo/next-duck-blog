# SEO 감사 가이드 — 이 블로그 기준

기존 글 또는 새 글의 SEO 이슈를 탐지하고 개선 방법을 제시하기 위한 체크리스트.

---

## 감사 절차

1. posts.json 또는 draft 파일에서 대상 글 확인
2. 아래 체크리스트 순서대로 이슈 탐지
3. 이슈별 우선순위(P1~P3) 부여
4. 수정 권고사항 before/after 형태로 제시

---

## 체크리스트

### P1 — 즉시 수정 (클릭률·색인에 직접 영향)

**title**
- [ ] 40~60자 범위인가? (너무 짧으면 정보 부족, 70자 초과면 잘림)
- [ ] 핵심 키워드가 앞 20자 안에 있는가?
- [ ] "~에 대해", "~을 알아보겠습니다" 같은 AI 교과서 톤인가?
- [ ] 블로그명이 title에 중복 포함됐는가? (Next.js template이 자동 추가하므로 불필요)

**summary**
- [ ] 비어있거나 1줄 이하인가?
- [ ] 80자 미만으로 너무 짧은가?
- [ ] 160자 초과로 snippet에서 잘릴 수 있는가?
- [ ] "소개합니다", "알아보겠습니다" 등 클릭 유인이 없는 표현인가?

**slug**
- [ ] 한글 slug인가? → 영문 또는 한글-영문 혼합 권장
- [ ] 너무 길거나 의미 없는 단어가 포함됐는가?

---

### P2 — 개선 권장 (검색 노출 품질에 영향)

**content 내 키워드 활용**
- [ ] title의 핵심 키워드가 본문 첫 100자 안에 등장하는가?
- [ ] 키워드가 본문에 자연스럽게 2~3회 이상 반복되는가?
- [ ] H2/H3 헤딩에 검색 키워드가 포함됐는가?

**tags**
- [ ] 태그가 3개 미만인가? (최소 3~5개 권장)
- [ ] 너무 일반적인 태그만 있는가? (`web`, `react` 단독 → 구체적인 태그 추가)

**images**
- [ ] images 필드가 비어있는가? (OG 이미지 없으면 소셜 공유 시 기본 배너만 표시)
- [ ] 글 내 첫 번째 이미지를 images[0]에 등록했는가?

---

### P3 — 선택적 개선 (장기적 SEO 강화)

**본문 구조**
- [ ] 글이 내부 링크 없이 독립적으로 존재하는가?
  - 연관 글(`http2-inspection-guide` ↔ `http2-practical-guide-part2` 등) 상호 링크 권장
- [ ] 코드 예제가 있는 기술 글에 `pre`, `code` 구조가 명확한가?

**date / lastmod**
- [ ] 오래된 글(`2021~2022`)의 내용이 현재도 유효한가?
  - 유효하면 `lastmod`를 현재 날짜로 업데이트 → Google 재크롤링 유도

---

## 감사 출력 형식

감사 완료 후 다음 형식으로 리포트:

```
## SEO 감사 결과: {slug}

### P1 이슈
- [title] "~에 대해 알아보겠습니다" → 키워드 누락, 교과서 톤
  권고: "JavaScript Proxy 패턴 완전 정복 — Vue 3 반응형의 비밀"

- [summary] 비어있음
  권고: "Vue 3의 반응형 시스템 핵심인 Proxy 패턴을 설명합니다. 
         GoF 디자인 패턴부터 ES6 도입 배경, 실전 활용까지 정리했습니다."

### P2 이슈
- [tags] ["javascript", "proxy"] → 너무 일반적
  권고: ["JavaScript Proxy", "Vue 3 반응형", "메타프로그래밍", "ES6", "Reflect API"]

### P3 이슈
- [lastmod] 2023-01-15 → 내용이 최신이면 업데이트 권장

변경률 예상: 5% 이하 (메타 필드만 수정)
```

## 빠른 감사 — 전체 글 일괄 확인

posts.json에서 다음 기준으로 이슈 글 목록 추출:

```javascript
// title이 너무 짧거나 AI 톤인 글
posts.filter(p => 
  p.title.length < 15 || 
  p.title.includes('알아보겠습니다') ||
  p.title.includes('에 대해')
)

// summary가 비거나 너무 짧은 글
posts.filter(p => !p.summary || p.summary.length < 50)

// tags가 부족한 글
posts.filter(p => !p.tags || p.tags.length < 3)
```
